szymixsiorek API

Cloudflare Workers - Production API

Wszystkie endpoints wymagaja naglowka: X-API-Key: chujcinamatke

O API

To jest prywatne API dostepne tylko z odpowiednim kluczem autoryzacyjnym. API obsluguje operacje CRUD na danych przechowywanych w Cloudflare KV Storage.

  • Autoryzacja poprzez X-API-Key header
  • Przechowywanie danych w KV Storage
  • Obsluga GET, POST, PUT, DELETE
  • Zwracanie JSON
  • Opcjonalny custom ID przy dodawaniu

Info Endpoints

GET /v1

Glowny endpoint API - zwraca przejrzysty overview wszystkich operacji

Przyklad:

curl https://api.szymixsiorek.workers.dev/v1 -H "X-API-Key: chujcinamatke"

Odpowiedz:

================================================================= szymixsiorek API v1.0 - OVERVIEW ================================================================= Przejrzysty overview z lista wszystkich dostepnych operacji
GET /v1/info

Informacje o workerze i jego konfiguracji

Przyklad:

curl https://api.szymixsiorek.workers.dev/v1/info -H "X-API-Key: chujcinamatke"
GET /v1/test

Test endpoint - sprawdzenie czy API jest dostepne i czy klucz jest prawidlowy

Przyklad:

curl https://api.szymixsiorek.workers.dev/v1/test -H "X-API-Key: chujcinamatke"

CRUD Operacje

GET /v1/data

Pobiera wszystkie elementy z bazy danych

Przyklad:

curl https://api.szymixsiorek.workers.dev/v1/data -H "X-API-Key: chujcinamatke"

Odpowiedz:

{ "status": "ok", "data": {...}, "count": 5 }
POST /v1/data

Dodaje nowy element. Wymagane: name. Opcjonalnie: id, description, value

Custom ID: Mozesz podac wlasne ID w polu "id". Jesli ID jest zajete, API automatycznie przydzieli nearest available ID.

Przyklad 1: Bez custom ID (auto-generate)

curl -X POST https://api.szymixsiorek.workers.dev/v1/data -H "X-API-Key: chujcinamatke" -H "Content-Type: application/json" -d "{\"name\":\"Test Element\",\"description\":\"Opis\",\"value\":123}"

Przyklad 2: Z custom ID

curl -X POST https://api.szymixsiorek.workers.dev/v1/data -H "X-API-Key: chujcinamatke" -H "Content-Type: application/json" -d "{\"name\":\"Test\",\"id\":\"100\"}"

Przyklad 3: Custom ID zajety - API zwroci wolny numer

curl -X POST https://api.szymixsiorek.workers.dev/v1/data -H "X-API-Key: chujcinamatke" -H "Content-Type: application/json" -d "{\"name\":\"Test\",\"id\":\"100\"}"

Jesli 100 jest zajete, otrzymasz: "Requested ID (100) is taken. Using ID: 101"

GET /v1/data/{id}

Pobiera konkretny element po ID

Przyklad:

curl https://api.szymixsiorek.workers.dev/v1/data/100 -H "X-API-Key: chujcinamatke"

Odpowiedz:

{ "status": "ok", "item": { "id": "100", "name": "Test", ... } }
PUT /v1/data/{id}

Edytuje istniejacy element. Mozesz zaktualizowac: name, description, value

Przyklad 1: Zmiana nazwy i wartosci

curl -X PUT https://api.szymixsiorek.workers.dev/v1/data/100 -H "X-API-Key: chujcinamatke" -H "Content-Type: application/json" -d "{\"name\":\"Nowa nazwa\",\"value\":999}"

Przyklad 2: Zmiana opisu

curl -X PUT https://api.szymixsiorek.workers.dev/v1/data/100 -H "X-API-Key: chujcinamatke" -H "Content-Type: application/json" -d "{\"description\":\"Nowy opis\"}"
DELETE /v1/data/{id}

Usuwa element z bazy danych

Przyklad:

curl -X DELETE https://api.szymixsiorek.workers.dev/v1/data/100 -H "X-API-Key: chujcinamatke"

Odpowiedz:

{ "status": "ok", "message": "Element usuniety", "deleted_item": {...} }

Struktura Danych

Kazdy element ma nastepujaca strukture:

{ "id": "123", "name": "string", "description": "string", "value": any, "created_at": "2025-11-15T15:00:00.000Z", "updated_at": "2025-11-15T15:00:00.000Z" }

Kody Odpowiedzi HTTP

200 OK - Request zakonczony pomyslnie
201 Created - Element zostal utworzony
400 Bad Request - Blad w danych (np. brakuje wymaganego pola)
401 Unauthorized - Zly/brak klucza API
404 Not Found - Zasob nie istnieje
500 Server Error - Blad wewnetrzny serwera
